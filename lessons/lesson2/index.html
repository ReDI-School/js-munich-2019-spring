Lesson 2: More Functions

Functions

So there is nothing wrong with

alert("Hello world!");


BUT you will never have 1 line of JS 😁

Let's say I want to find the distance a space ship travels

------------> 🚀

  • distance = speed * time
  • In JavaScript:
  • let speed = 10;
let time = 5;
alert(speed * time);

And if I have more values?

let speed1 = 56;
let time1 = 7;
alert(speed1 * time1);

let speed2 = 12;
let time2 = 3;
alert(speed2 * time2);

let speed3 = 9;
let time3 = 88;
alert(speed3 * time3);

let speed4 = 64;
let time4 = 90;
alert(speed4 * time4);

Awful! 😫

So much repeated code!

Solve it by Function!

function showDistance (speed, time) {
  alert(speed * time);
}

showDistance(56, 7);
showDistance(12, 3);
showDistance(9, 88);
showDistance(64, 90);

Definition of a function

  • Group statements together
  • Make your code reuseable

What is an argument?

Remember alert("Hello!") ?

Or showDistance(speed,time) ?

arguments are between the () separated by ,


function showDistance(speed, time) {
  alert(speed * time);
}
showDistance(56,7);

Function call: order matters

Function arguments are treated like variables inside the function

Dive in:

  • Code a function named askName
  • Prompts the user to "Enter name"
  • Then assigns the answer to userName, which hasn't been declared beforehand
  • Then use alert to show the name.
  • Don't forget to make the function call.

function askName() {
  let userName = prompt("Enter name");
  alert(userName);
}
askName();

Dive in:

  • Code a function named addStrings
  • Takes 2 arguments
  • Combine the 2 strings and store it
  • Then show the the stored Variable in an alert box
  • Don't forget to make the function call.

function addStrings(string1, string2) {
  let combine = string1 + string2;
  alert(combine);
}
addStrings("One", "Apple");
